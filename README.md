VIRTUAL CLINIX — Telemedicine Platform
Overview
VIRTUAL CLINIX is a web-based telemedicine platform designed to connect patients with verified medical specialists for remote consultations. The platform supports the full consultation lifecycle — from doctor discovery and appointment booking, through secure video consultation, to automated medical record generation and e-prescription delivery. It is built entirely on a zero-cost technology stack, making it suitable for prototype deployment without any monthly infrastructure spend.
Architecture
The platform follows a 3-Tier Architecture separating the system into three independently scalable layers. The Presentation Layer is built with React.js, TailwindCSS, and Redux, serving three role-specific portals for patients, doctors, and administrators from a single codebase. The Application Layer runs entirely on Supabase Edge Functions, ensuring that client applications never communicate with the database directly — all business logic, authentication, and access control are enforced server-side. The Data Layer uses Supabase PostgreSQL with Row-Level Security policies enforced on every table, guaranteeing that data access is controlled at the database level independently of application code.
Security
Security is implemented as a defence-in-depth model across six independent layers. All data in transit is protected by TLS 1.3. Authentication is handled via Supabase Auth with JWT tokens and optional TOTP-based two-factor authentication for doctors and administrators. Authorisation is enforced through PostgreSQL Row-Level Security — a doctor can only query records for patients with whom they have an existing appointment, and unverified doctors are invisible to patients entirely. Every read and write operation is appended to an immutable audit_logs table. Prescriptions are stored as structured JSONB alongside a SHA-256 hash for tamper detection, and payment confirmation is processed exclusively via server-side gateway webhooks.
Getting Started
Clone the repository, install dependencies with npm install, configure your environment variables by copying .env.example to .env, and run npm run dev to start the development server. Supabase migrations can be applied with npx supabase db push. Full setup instructions are documented in the /docs folder.
Technology Stack
The entire platform operates at zero monthly cost during prototype phase. The frontend is hosted on Vercel, backend logic runs on Supabase Edge Functions, video consultations are powered by Jitsi Meet with no API key requirement, transactional emails are delivered via Resend.com, and all data is stored in Supabase PostgreSQL with 500MB of free storage. The upgrade path to a production-scale deployment requires only a Supabase Pro subscription at $25/month — no architectural changes are needed.
Status
This project is currently in active prototype development. The core booking flow, doctor verification pipeline, video consultation module, and medical record generation are implemented. Payment gateway integration is configured in test mode. Contributions, issues, and feature suggestions are welcome via the repository's issue tracker.
